Given a company's subsidiaries (in HTML, Markdown, or plain text previously
extracted from a PDF), return JSON in this shape:

```json
{
    "subsidiaries": [
        {"name": "XXX", "location": "YYY", "source_quote": "ZZZ"}
    ]
}
```

Fields:

- name: the subsidiary name. If the input is HTML, decode HTML entities
(e.g. "PF&amp;F Holdings Corporation" → "PF&F Holdings Corporation").
Otherwise copy the name exactly as it appears. Do not expand or drop
abbreviations (keep "Co.", "Inc.", "Ltd."). Do not prepend section
headers or geographic labels — if a "Canada" header precedes
"U-Haul Co. (Canada) Ltd.", the name is "U-Haul Co. (Canada) Ltd.",
not "Canada: U-Haul Co. (Canada) Ltd.".

- location: the place of incorporation or other location, or null if not
provided.

- source_quote: a short verbatim snippet from the input text that
contains the subsidiary's name. Must be an exact substring of the
provided text, byte-for-byte — do not paraphrase, re-capitalize, or
decode HTML entities in this field. If you cannot find such a snippet,
omit the subsidiary entirely. If a subsidiary appears multiple times,
pick the snippet with the most surrounding context.

Include all subsidiaries. If the document groups subsidiaries under parents
or categories, flatten and return every named entity as its own row,
regardless of hierarchy.
